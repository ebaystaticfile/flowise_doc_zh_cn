# 贡献指南

***

我们感谢所有贡献！无论您的技能水平或技术背景如何，您都可以帮助这个项目发展壮大。以下是一些贡献方式：

## ⭐ Star（加星）

为 [Flowise Github 仓库](https://github.com/FlowiseAI/Flowise) 加星并分享。

## 🙌 分享 Chatflow

是的！分享您如何使用 Flowise 也是一种贡献方式。将您的 chatflow 导出为 JSON 格式，附加截图，并在 [展示与讨论区](https://github.com/FlowiseAI/Flowise/discussions/categories/show-and-tell) 分享。

## 💡 建议

我们欢迎您提出新功能和应用集成的建议。请将您的建议提交到 [建议区](https://github.com/FlowiseAI/Flowise/discussions/categories/ideas)。

## 🙋 Q\&A（问答）

想了解更多信息？请在 [问答区](https://github.com/FlowiseAI/Flowise/discussions/categories/q-a) 搜索问题的答案。如果您找不到答案，请随时提出新的问题。这可能会帮助其他有类似问题的人。

## 🐞 报告 Bug

发现问题了吗？[提交 Bug 报告](https://github.com/FlowiseAI/Flowise/issues/new/choose)。

## 📖 贡献文档

1. Fork 官方的 [Flowise 文档仓库](https://github.com/FlowiseAI/FlowiseDocs)
2. 克隆您 fork 的仓库
3. 创建一个新的分支
4. 切换到您刚刚创建的分支
5. 进入仓库文件夹

    代码块 0 bash
    ```bash
    cd FlowiseDocs
    ```
6. 进行修改
7. 提交更改并从 fork 的分支提交 Pull Request 到 [FlowiseDocs 主分支](https://github.com/FlowiseAI/FlowiseDocs)

## 👨‍💻 贡献代码

要了解如何贡献代码，请访问 [开发者指南](../getting-started/#setup-2) 部分并按照说明操作。

如果您正在贡献新的节点集成，请阅读 [构建节点](building-node_zh.md) 指南。

## 🏷️ Pull Request 流程

当您打开 Pull Request 时，FlowiseAI 团队成员将自动收到通知/指派。您也可以在 [Discord](https://discord.gg/jbaHfsRVBW) 上联系我们。

## 📜 行为准则

本项目及其所有参与者均受行为准则约束，该准则可在 [此处](https://github.com/FlowiseAI/Flowise/blob/main/CODE_OF_CONDUCT_zh.md) 查看。参与本项目即表示您同意遵守此准则。

请将不可接受的行为报告至 hello@flowiseai.com。
